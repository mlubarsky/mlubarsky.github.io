Carbon in Context is a project that helps users understand greenhouse gas emissions through everyday comparisons, with the additional goal of eliminating greenwashing and misinformation.

  • Features real-world metaphors including "miles driven" and "hours of netflix watched".
  • Implements a cloropleth map of the USA with CO2 emissions per state.
  • Provides access to Project Drawdown's climate solutions.
  • Allows open access to emissions data.

This search engine was created from scratch in 2000 source-lines of code (SLOC), and utilizes Apache OpenNLP, TD-IDF results rankings, Apache Log4j2, sockets, HTTP, regular expressions, Eclipse Jetty, servlets, Bulma, JUnit, and Apache Maven.

  • Backend written in Java, Frontend written in HTML/CSS, JavaScript.
  • Implements a custom inverted index data structure.
  • Multithreaded web crawler supporting thousands of pages.

This predictive time series model uses a Long Short-Term Memory (LSTM) recurrent neural network to predict San Francisco’s AQI based on historical weather and pollution data.

  • Built with ML libraries including TensorFlow, Scikit-learn, Pandas, Numpy, Matplotlib.
  • Achieved an 87% prediction accuracy after training and validation.
  • Visualizes real vs. predicted AQI values using Matplotlib.

This algorithm was created from scratch, then trained and evaluated on blobs and weather datasets.

  • Built in Jupyter Notebook using Python.
  • Achieved a 97.5% clustering accuracy, which is on par with Scikit-learn's implementation.

Implemented client/server and standalone applications which utilize network sockets to send packets between both machines, and perform a calculation that shows whether there is compression on a network link.

  • All 3 applications written in C, utilizing WireShark for packet inspection and GDB for debugging.
  • Installed Linux images on Oracle VM VirtualBox machines, which were used as client/server.

Created an emulator for the RISC-V instruction set architecture that can perform various assembly instructions, analyze the instructions used in a emulated C program.

  • Written in C and used GDB for debugging.
